## Modes

### Sandbox Mode (default)
- Runs a full OpenClaw instance entirely in your browser
- Requires an Anthropic API key (or OpenRouter key)
- Key stored in localStorage â€” never sent to VibeClaw servers
- Ideal for: trying OpenClaw, testing prompts, quick prototyping, demos

### Live Gateway Mode
- Connects to your real running OpenClaw gateway via WebSocket
- Requires a running OpenClaw installation and a gateway token
- No API key needed (gateway handles auth)
- Full dashboard: sessions, agents, files, skills, cron jobs, cost tracking, live logs
- Default gateway: localhost:18789 (via Vite proxy)
- Ideal for: monitoring your production OpenClaw, remote management

## FAQ

**Q: Is my API key safe?**
A: Yes. In sandbox mode, your API key stays in your browser and goes directly to Anthropic's API. It's never sent to VibeClaw servers. It's stored in localStorage â€” clear it anytime. In live mode, no API key is needed.

**Q: What's the difference between Sandbox and Live mode?**
A: Sandbox boots an isolated OpenClaw container in your browser â€” just paste your API key and start chatting. Live connects to your actual running OpenClaw gateway via WebSocket for the full production dashboard.

**Q: How does the sandbox actually work?**
A: VibeClaw uses almostnode â€” a browser-native Node.js runtime with a virtual filesystem, 40+ shimmed modules, and npm support. It creates a real container, loads the OpenClaw VFS snapshot (67 files), and bootstraps the full runtime. Chat calls go direct to the Anthropic API from your browser.

**Q: What can I do in live gateway mode?**
A: Everything. The dashboard shows all sessions with token usage bars, agent identity and workspace files (SOUL.md, MEMORY.md, etc.), active and available skills, cron jobs with status, daily cost sparklines, connected nodes, gateway logs, and full streaming chat.

**Q: Do I need to install anything?**
A: No. For sandbox mode, just open the page and paste your Anthropic API key. For live mode, you need a running OpenClaw gateway.

**Q: What are Flavours?**
A: Flavours are preconfigured sandbox personalities. Each flavour defines its own agents, skills, system prompt, and workspace layout. The default ðŸ¦€ OpenClaw is a coding assistant. ðŸ¦ž TinyClaw is a multi-agent orchestrator.

**Q: What about WebGPU / local models?**
A: Coming soon. We're integrating WebLLM to run Qwen2.5-Coder 1.5B (4-bit quantized, ~900MB) entirely via WebGPU. One download, then fully offline â€” no API key, no network. The Clawdio widget already runs Qwen2.5-1.5B via WebGPU today.

**Q: Can I use this in production?**
A: The sandbox is great for trying OpenClaw, testing prompts, and prototyping. For production workloads, run a real OpenClaw gateway and use live mode to monitor and chat with it from anywhere.

**Q: What models are supported?**
A: Any model available via OpenRouter (free and paid). Direct Anthropic API keys also work. The TinyClaw widget supports WebGPU local models including Qwen2.5-1.5B (~1GB) with no API key required.
